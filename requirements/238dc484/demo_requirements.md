# Requirements: Parking Lot Management System

## Functional Requirements

### FR1: Vehicle Entry
The system shall record vehicle details at entry.

### FR2: Vehicle Entry
The system shall generate a parking ticket.

### FR3: Vehicle Entry
The system shall assign an available parking slot.

### FR4: Slot Management
The system shall display available parking slots.

### FR5: Slot Management
The system shall update slot status (available/occupied).

### FR6: Slot Management
The admin shall be able to add or remove parking slots.

### FR7: Vehicle Exit
The system shall record vehicle exit time.

### FR8: Vehicle Exit
The system shall calculate parking duration.

### FR9: Vehicle Exit
The system shall generate the parking fee.

### FR10: Payment Management
The system shall allow payment calculation.

### FR11: Payment Management
The system shall generate payment receipts.

### FR12: Report Generation
The system shall generate reports for: Daily parking records

### FR13: Report Generation
The system shall generate reports for: Total revenue

### FR14: Report Generation
The system shall generate reports for: Slot utilization

### FR15: User Management
The admin shall manage staff accounts.

### FR16: User Management
The system shall authenticate users before login.

## Non-Functional Requirements

- **NFR1**: The system should process entry and exit requests within 2 seconds.
- **NFR2**: User authentication should be required for admin access.
- **NFR3**: Parking data should be stored securely.
- **NFR4**: The system should be available 24/7.
- **NFR5**: Data should be backed up regularly.
- **NFR6**: The interface should be simple and easy to use.
