# User Stories: Movie Hall Management System

**Total User Stories:** 10
**Estimated Effort:** 16 days

## Sprint Week 1: User Authentication

### US1: User Registration
The system shall allow customers to register using personal details such as name, email, and password.

**Acceptance Criteria:**
- A user can register with a unique email and password.
- User details are stored securely in the database.

**Linked FRs:** FR1

### US2: User Login
The system shall allow registered users to log in securely.

**Acceptance Criteria:**
- A user can log in using their email and password.
- Secure authentication is implemented.

**Linked FRs:** FR2

## Sprint Week 2: Admin Functions

### US3: Movie Management
The administrator shall be able to add, update, and remove movies from the system.

**Acceptance Criteria:**
- Admin can add new movies with details.
- Admin can update movie details.
- Admin can remove movies from the system.

**Linked FRs:** FR3

### US4: Show Scheduling
The administrator shall schedule movie showtimes and assign them to specific halls.

**Acceptance Criteria:**
- Admin can schedule new shows.
- Shows are assigned to specific halls.

**Linked FRs:** FR4

## Sprint Week 3: Customer Functions

### US5: View Movies
Customers shall be able to view movie listings, details, and available showtimes.

**Acceptance Criteria:**
- Customers can view a list of movies.
- Customers can view movie details and available showtimes.

**Linked FRs:** FR5

### US6: Seat Selection
The system shall allow customers to select available seats for a movie show.

**Acceptance Criteria:**
- Customers can select available seats.
- Selected seats are marked as booked.

**Linked FRs:** FR6

## Sprint Week 4: Final Features

### US7: Ticket Booking
Customers shall be able to book tickets for selected seats.

**Acceptance Criteria:**
- Customers can book tickets for selected seats.
- Bookings are confirmed.

**Linked FRs:** FR7

### US8: Payment Processing
The system shall allow customers to make payments for booked tickets.

**Acceptance Criteria:**
- Customers can make payments for booked tickets.
- Payments are processed securely.

**Linked FRs:** FR8

### US9: Ticket Generation
The system shall generate a digital ticket after successful booking.

**Acceptance Criteria:**
- Digital tickets are generated after successful booking.
- Tickets are sent to the customer.

**Linked FRs:** FR9

### US10: Booking History
Customers shall be able to view their booking history.

**Acceptance Criteria:**
- Customers can view their booking history.
- History includes all past bookings.

**Linked FRs:** FR10
