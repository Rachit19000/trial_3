# User Stories: Parking Lot Management System

**Total User Stories:** 14
**Estimated Effort:** 28 days

## Sprint Week 1: Sprint 1

### US001: Register vehicles entering the parking lot
The system should be able to record vehicle details at entry.

**Acceptance Criteria:**
- Vehicle details are recorded accurately.
- Vehicle entry is logged in the system.

**Linked FRs:** 3.1.1

### US002: Generate a parking ticket
The system should generate a parking ticket upon vehicle entry.

**Acceptance Criteria:**
- A unique ticket ID is generated.
- Ticket is linked to the vehicle entry.

**Linked FRs:** 3.1.2

### US003: Automatically assign an available parking slot
The system should automatically assign an available parking slot to the vehicle.

**Acceptance Criteria:**
- The nearest available slot is assigned.
- Slot is marked as occupied in the system.

**Linked FRs:** 3.1.3

## Sprint Week 2: Sprint 2

### US004: Display available parking slots

**Acceptance Criteria:**

- Slots are updated in real-time.

**Linked FRs:** 3.2.1

### US005: Update slot status (available/occupied)
The system should update the status of parking slots as they become available or occupied.

**Acceptance Criteria:**
- Slot status is updated accurately.
- Status is reflected in real-time.

**Linked FRs:** 3.2.2

### US006: Add or remove parking slots
The admin should be able to add or remove parking slots.

**Acceptance Criteria:**

- Changes are reflected in the system.

**Linked FRs:** 3.2.3
