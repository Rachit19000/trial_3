# User Stories: Movie Hall Management System

**Total User Stories:** 10
**Estimated Effort:** 20 days

## Sprint Week 1: User Authentication

### US1: User Registration
The system shall allow customers to register using personal details such as name, email, and password.

**Acceptance Criteria:**
- Users can register with valid personal details.
- User details are stored securely.
- Duplicate registrations are not allowed.

**Linked FRs:** FR1

### US2: User Login
The system shall allow registered users to log in securely.

**Acceptance Criteria:**
- Users can log in with valid credentials.
- Login process is secure.
- Invalid credentials result in appropriate error messages.

**Linked FRs:** FR2

## Sprint Week 2: Admin Functions

### US3: Movie Management
The administrator shall be able to add new movies, update movie details, and remove movies from the system.

**Acceptance Criteria:**
- Administrators can add new movies.
- Administrators can update movie details.
- Administrators can remove movies from the system.

**Linked FRs:** FR3

### US4: Show Scheduling
The administrator shall schedule movie showtimes and assign them to specific halls.

**Acceptance Criteria:**
- Administrators can schedule movie showtimes.
- Showtimes are assigned to specific halls.
- Showtimes are displayed correctly.

**Linked FRs:** FR4

## Sprint Week 3: Customer Functions

### US5: View Movies
Customers shall be able to view movie list, movie details, and available showtimes.

**Acceptance Criteria:**
- Customers can view movie list.
- Customers can view movie details.
- Customers can view available showtimes.

**Linked FRs:** FR5

### US6: Seat Selection
The system shall allow customers to select available seats for a movie show.

**Acceptance Criteria:**
- Customers can select available seats.
- Selected seats are marked as booked.
- Customers cannot select already booked seats.

**Linked FRs:** FR6

## Sprint Week 4: Final Features

### US7: Ticket Booking
Customers shall be able to book tickets for selected seats.

**Acceptance Criteria:**
- Customers can book tickets for selected seats.
- Bookings are confirmed.
- Bookings are stored in the system.

**Linked FRs:** FR7

### US8: Payment Processing
The system shall allow customers to make payments for booked tickets.

**Acceptance Criteria:**
- Customers can make payments for booked tickets.
- Payments are processed securely.
- Payments are stored in the system.

**Linked FRs:** FR8

### US9: Ticket Generation
The system shall generate a digital ticket after successful booking.

**Acceptance Criteria:**
- Digital tickets are generated after successful booking.
- Tickets are stored in the system.
- Tickets are accessible to customers.

**Linked FRs:** FR9

### US10: Booking History
Customers shall be able to view their booking history.

**Acceptance Criteria:**
- Customers can view their booking history.
- Booking history is accurate.
- Booking history is stored in the system.

**Linked FRs:** FR10
